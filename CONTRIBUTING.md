    # Guide de contribution

## Règles de base

- **Jamais de push direct sur `main`.**
- Toute modification passe par une Pull Request.
- Une PR doit avoir **au moins 1 approbation** avant le merge.

## Convention de nommage des branches

| Type           | Exemple                  |
| -------------- | ------------------------ |
| Fonctionnalité | `feat/nom-de-la-feature` |
| Correctif      | `fix/nom-du-bug`         |
| Documentation  | `docs/nom-du-sujet`      |

## Ouvrir une Pull Request

1. Crée ta branche depuis `main` à jour
2. Fais tes commits (convention : `feat:`, `fix:`, `chore:`, `docs:`)
3. Pousse ta branche et ouvre une PR vers `main`
4. Décris : **quoi**, **pourquoi**, **comment tester**
5. Demande une revue à un coéquipier
